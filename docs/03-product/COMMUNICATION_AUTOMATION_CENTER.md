# ADR-006 — Unified Communication Automation Center

## Context

SHIK Platform должна отправлять клиентам, сотрудникам, управляющим и владельцам сообщения через различные каналы.

Поддерживаемые каналы:

- Push Notifications
- In-App Messages
- WhatsApp Business
- Telegram
- Email
- SMS
- Webhooks

Использование одного фиксированного канала ограничило бы владельца ресторана и создало зависимость от конкретного провайдера.

## Decision

Создать единый модуль Communication Automation Center.

Владелец или уполномоченный менеджер сможет через Back Office определять:

- какие события создают сообщения;
- каким получателям они отправляются;
- через какие каналы;
- с какой задержкой;
- с каким шаблоном;
- какие резервные каналы используются при недоставке;
- какие правила действуют для конкретного филиала.

## Scope

### MVP

- включение и отключение каналов;
- правила коммуникаций по событиям заказа;
- Push;
- In-App;
- Email;
- WhatsApp Business;
- Telegram для внутренних уведомлений;
- шаблоны сообщений;
- журнал доставки;
- настройки по филиалам;
- согласия клиентов;
- тихие часы для маркетинговых сообщений.

### Phase 2

- цепочки сообщений;
- резервные каналы;
- сегментация клиентов;
- кампании;
- аналитика каналов;
- A/B-тестирование;
- сложные условные правила.

## Key Events

- OrderCreated
- OrderConfirmed
- OrderPreparing
- OrderReady
- CourierAssigned
- OrderOutForDelivery
- OrderCompleted
- OrderCancelled
- PaymentFailed
- LoyaltyPointsAdded
- LowInventoryDetected
- ShiftClosed
- SuspiciousRefundDetected

## Rules

1. Операционные сообщения о текущем заказе могут отправляться независимо от маркетинговой подписки, если это разрешено законом и политикой платформы.
2. Рекламные сообщения отправляются только при наличии соответствующего согласия клиента.
3. Клиент может управлять разрешенными каналами в своем профиле.
4. Владелец выбирает каналы по умолчанию через Back Office.
5. Настройки клиента имеют приоритет над маркетинговыми настройками владельца.
6. Критические системные уведомления не должны использовать маркетинговые шаблоны.
7. Все попытки отправки фиксируются в журнале.
8. Интеграции с каналами реализуются через независимые адаптеры.

## Consequences

### Positive

- владелец самостоятельно управляет коммуникациями;
- добавление нового канала не меняет бизнес-логику;
- возможны настройки по филиалам;
- появляется единая история коммуникаций;
- уменьшается зависимость от одного поставщика.

### Risks

- стоимость сообщений внешних провайдеров;
- требования к согласию пользователей;
- ограничения шаблонов WhatsApp Business;
- необходимость контроля повторных сообщений;
- риск отправки избыточного количества уведомлений.
